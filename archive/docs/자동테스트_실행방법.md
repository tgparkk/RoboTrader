# 장중 자동 테스트 실행 방법

## 개요
- **목적**: 오늘(2026-01-27 화요일) 장중에 자동으로 분봉 API 테스트
- **실행 시간**: 10:30, 12:00, 14:00 (3회 자동 실행)
- **독립 실행**: 봇과 별개로 실행 가능

## 실행 방법

### 방법 1: 터미널에서 직접 실행 (권장)
```bash
# 새 터미널 열기
cd D:\GIT\RoboTrader
python auto_test_minute_api.py
```

**실행 후:**
- 다음 테스트 시간까지 자동 대기
- 10:30, 12:00, 14:00에 자동으로 테스트 실행
- 장 마감 후 자동 종료

### 방법 2: 백그라운드 실행 (Windows)
```bash
# 새 터미널에서 실행 후 최소화
cd D:\GIT\RoboTrader
start python auto_test_minute_api.py
```

### 방법 3: Task Scheduler 등록 (고급)
```powershell
# 오전 9시에 자동 시작
schtasks /create /tn "RoboTrader_API_Test" /tr "python D:\GIT\RoboTrader\auto_test_minute_api.py" /sc once /st 09:00 /sd 01/27/2026
```

## 특징

### 자동 스케줄링
- **10:30**: 충분한 과거 데이터 확보 (1시간 30분 거래)
- **12:00**: 오전 거래 마무리 시점
- **14:00**: 오후 거래 중반

### 안전 기능
- ✅ 날짜 검증: 2026-01-27만 실행
- ✅ 장중 시간 확인: 09:00~15:30만 실행
- ✅ 중복 실행 방지: 같은 시간대 재실행 방지
- ✅ 봇 독립 실행: API 토큰 공유, 영향 없음

### 수동 중단
- `Ctrl+C`: 언제든지 안전하게 종료

## 봇과 동시 실행

### 안전성
- ✅ **API 토큰 공유**: token_info.json 파일 사용
- ✅ **API 제한**: 초당 20건 (봇 + 테스트 합산 충분)
- ✅ **독립 실행**: 봇 프로세스에 영향 없음

### 모니터링
```bash

```

## 문제 해결

### API 인증 실패
```
[ERROR] API 인증 실패
```
→ config/key.ini 파일 확인 또는 봇이 실행 중인지 확인

### 날짜 불일치
```
[종료] 오늘은 테스트 실행일(2026-01-27)이 아닙니다.
```
→ 정상 동작 (다른 날짜는 실행 안 됨)

### 장 마감 후 실행
```
[종료] 장 마감 시간입니다. 테스트를 종료합니다.
```
→ 정상 동작 (15:30 이후 자동 종료)
