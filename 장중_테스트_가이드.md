# 장중 분봉 API 테스트 가이드

## 테스트 목적
`past_data_yn="Y"` vs `"N"` 옵션의 실제 차이를 장중에 확인

## 테스트 방법

### 방법 1: 봇 실행 중에도 테스트 가능 (권장)
```bash
# 별도 터미널에서 실행
cd D:\GIT\RoboTrader
python test_minute_api_response.py
```

**장점:**
- 봇과 독립적으로 실행 가능
- API 토큰은 공유되므로 추가 인증 불필요
- 봇 실행에 영향 없음

**주의사항:**
- KIS API 속도 제한: 초당 20건 (봇 + 테스트 합산)
- 테스트는 2번의 API 호출만 수행하므로 영향 미미

### 방법 2: 봇 중지 후 테스트
```bash
# 1. 봇 중지 (실행 중인 경우)
# Ctrl+C 또는 kill 명령

# 2. 테스트 실행
cd D:\GIT\RoboTrader
python test_minute_api_response.py

# 3. 봇 재시작
python main.py
```

## 예상 결과

### 장 마감 후 (현재 상태)
```
past_data_yn='Y': 30건
past_data_yn='N': 30건
차이: 0건
```

### 장중 (09:00~15:30)
```
past_data_yn='Y': 최대 30건 (과거 데이터 포함)
past_data_yn='N': 1~몇 건 (요청 시점 이후만)
```

**예시:**
- 현재 시간: 10:30
- `input_hour`: 10:28 (2분 전 완성된 분봉)
- `"Y"`: 09:59~10:28 (30개 분봉)
- `"N"`: 10:28~10:28 (1개 분봉, 또는 요청 이후만)

## 테스트 시간대 추천

### 최적 시간
- **10:00~14:30**: 충분한 과거 데이터 존재
- 이 시간대에는 30개 분봉 차이가 명확히 나타남

### 피해야 할 시간
- **09:00~09:30**: 과거 데이터 부족으로 차이가 작음
- **15:20~15:30**: 장 마감 임박으로 불안정

## 데이터 불일치 원인 (재확인)

1. **API 동작 방식**
   - `past_data_yn="Y"`: 매번 최대 30개 분봉 반환
   - API가 장중에 과거 데이터를 수정/업데이트할 수 있음

2. **데이터 병합 전략**
   - `drop_duplicates(keep='last')`: 나중 API 응답으로 덮어쓰기
   - 09:45 신호 시점 데이터 ≠ 15:34 메모리 덤프 데이터

3. **결론**
   - 현재 구현은 API의 최신 데이터를 신뢰하는 방식
   - 이는 합리적인 선택 (거래소 수정 데이터 반영)
